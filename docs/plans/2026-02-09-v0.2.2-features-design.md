# Koda v0.2.2 Features Design

**Date:** 2026-02-09
**Status:** Design Complete, Ready for Implementation
**Estimated Effort:** 5-6 hours

## Overview

Three key features for v0.2.2 to improve user experience:

1. **Manual Update Check Button** - Allow users to check for updates anytime, especially useful if stuck in setup
2. **Dynamic Model Selector** - Searchable model picker that gets list from `openclaw models list`, reusable in Auth and Dashboard
3. **Config Template Cache** - Save user configuration (provider, model, API key, Telegram info) to avoid re-entering

## Background

### Current Pain Points

1. **No manual update trigger** - Users stuck in early setup steps can't access Dashboard to see update banner
2. **Hardcoded model selection** - Can't adapt to new models added to OpenClaw
3. **Repeated configuration** - Users must re-enter all info if OpenClaw config gets corrupted

### Design Goals

1. Update check accessible from any setup step
2. Model list dynamically fetched from OpenClaw, always up-to-date
3. Configuration cached in app for quick recovery
4. No breaking changes to existing setup flow

## Component Specifications

### 1. ConfigTemplateCache

**Purpose:** Cache user configuration to avoid re-entering information.

**File:** `app/src/main/java/app/botdrop/ConfigTemplateCache.java`

**Storage:** SharedPreferences (no encryption needed - convenience cache)

**Fields:**
```java
public class ConfigTemplate {
    public String provider;      // "google"
    public String model;         // "gemini-3-flash-preview"
    public String apiKey;        // "AIzaSy..."
    public String tgBotToken;    // "7123456:AAF..." (optional)
    public String tgUserId;      // "987654321" (optional)

    public boolean isValid() {
        return provider != null && !provider.isEmpty()
            && model != null && !model.isEmpty()
            && apiKey != null && !apiKey.isEmpty();
    }
}
```

**Methods:**
```java
public class ConfigTemplateCache {
    private static final String PREFS_NAME = "botdrop_config_template";

    // Save configuration template
    public static void saveTemplate(Context ctx, ConfigTemplate template);

    // Load configuration template
    public static ConfigTemplate loadTemplate(Context ctx);

    // Clear cache
    public static void clearTemplate(Context ctx);

    // Check if cache exists
    public static boolean hasTemplate(Context ctx);

    // Apply template to OpenClaw configuration files
    public static boolean applyTemplate(Context ctx, ConfigTemplate template) {
        KodaConfig.setProvider(template.provider, template.model);
        KodaConfig.setApiKey(template.provider, template.apiKey);
        if (template.tgBotToken != null) {
            KodaConfig.setTelegramChannel(template.tgBotToken, template.tgUserId);
        }
        return true;
    }
}
```

**Error Handling:**
- Corrupted SharedPreferences → Clear and return null
- Missing fields → Validate with `isValid()`
- Version migration → Support config version upgrade

### 2. ModelSelectorDialog

**Purpose:** Searchable model selector that fetches list from OpenClaw.

**File:** `app/src/main/java/app/botdrop/ModelSelectorDialog.java`

**UI Layout:**
```xml
<LinearLayout vertical>
    <TextView text="Select Model" />
    <EditText hint="Search models..." /> <!-- Real-time filter -->
    <RecyclerView /> <!-- Model list -->
    <TextView id="status" /> <!-- Loading/Error message -->
</LinearLayout>
```

**Workflow:**
1. Dialog shown → Show loading state
2. Execute `openclaw models list` via KodaService
3. Parse output into ModelInfo list
4. Display in RecyclerView
5. User types in search box → Filter list in real-time
6. User clicks model → Callback `onModelSelected(provider, model)` → Close dialog

**Model Parsing:**

Input format:
```
Model                                      Input      Ctx      Local Auth  Tags
google/gemini-3-flash-preview              text+image 1024k    no    yes   default
anthropic/claude-sonnet-4-5                text+image 200k     no    yes
```

Parse to:
```java
public class ModelInfo {
    public String fullName;  // "google/gemini-3-flash-preview"
    public String provider;  // "google"
    public String model;     // "gemini-3-flash-preview"
    public String input;     // "text+image"
    public String context;   // "1024k"
    public boolean isDefault; // has "default" tag
}
```

**Search Logic:**
```java
private void filterModels(String query) {
    String lowerQuery = query.toLowerCase();
    List<ModelInfo> filtered = allModels.stream()
        .filter(m -> m.fullName.toLowerCase().contains(lowerQuery))
        .collect(Collectors.toList());
    adapter.updateList(filtered);
}
```

**Error Handling:**
- Command fails → Show error message + retry button
- No models found → Show "No models available"
- Parse error → Show default model list fallback

**Callback Interface:**
```java
public interface ModelSelectedCallback {
    void onModelSelected(String provider, String model);
}
```

### 3. Manual Update Button

**Purpose:** Allow users to check for updates from any setup step.

**Location:** SetupActivity navigation bar (visible across all steps)

**Layout Changes:**

`activity_botdrop_setup.xml`:
```xml
<LinearLayout android:id="@+id/setup_navigation" horizontal>
    <Button android:id="@+id/setup_button_back" text="Back" />

    <!-- NEW: Manual update button -->
    <ImageButton
        android:id="@+id/setup_check_updates"
        android:src="@drawable/ic_update"
        android:contentDescription="Check for updates"
        android:background="?attr/selectableItemBackgroundBorderless"
        android:padding="12dp" />

    <Space android:layout_weight="1" />
    <Button android:id="@+id/setup_button_next" text="Next" />
</LinearLayout>
```

**UpdateChecker Enhancement:**

Add force check method (ignore 24-hour throttle):
```java
// UpdateChecker.java
public static void forceCheck(Context ctx, UpdateCallback cb) {
    // Clear last check timestamp to force immediate check
    SharedPreferences prefs = ctx.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE);
    prefs.edit().putLong(KEY_LAST_CHECK, 0).apply();

    // Execute check
    check(ctx, cb);
}
```

**SetupActivity Integration:**
```java
// SetupActivity.onCreate()
Button checkUpdatesBtn = findViewById(R.id.setup_check_updates);
checkUpdatesBtn.setOnClickListener(v -> {
    v.setEnabled(false); // Prevent double-click
    UpdateChecker.forceCheck(this, (version, url, notes) -> {
        if (version != null) {
            showUpdateBanner(version, url);
        } else {
            Toast.makeText(this, "No updates available", Toast.LENGTH_SHORT).show();
        }
        v.setEnabled(true);
    });
});
```

```

    ↓
```

```
  → KodaConfig.setApiKey(...)

    ↓
```

### Flow 3: Model Change from Dashboard

```
User in Dashboard
    ↓
Clicks [Change Model]
    ↓

    ↓
User searches and selects new model
    ↓

    ↓

```

### Flow 4: Manual Update Check

```
User in SetupActivity (any step)
    ↓

    ↓
Button disabled (prevent double-click)
    ↓
UpdateChecker.forceCheck(context, callback)
  → Clear last check timestamp
  → Execute check immediately
    ↓
If update available:
  → Show update banner with download button
Else:
  → Show toast: "No updates available"
    ↓
Re-enable button
```

## Error Handling

### ModelSelectorDialog Errors

**Scenario: openclaw command fails**
```java
if (!result.success) {
    mStatusText.setText("Failed to load models");
    mRetryButton.setVisibility(View.VISIBLE);
    mRetryButton.setOnClickListener(v -> loadModels());
}
```

**Scenario: No models returned**
```java
if (modelList.isEmpty()) {
    mStatusText.setText("No models available. Configure auth first.");
}
```

**Scenario: Parse error**
```java
try {
    parseModelList(output);
} catch (Exception e) {
    Logger.logError(TAG, "Parse error: " + e);
    // Show default models as fallback
    showDefaultModels();
}

private void showDefaultModels() {
    List<ModelInfo> defaults = Arrays.asList(
        new ModelInfo("google/gemini-3-flash-preview", "google", "gemini-3-flash-preview"),
        new ModelInfo("anthropic/claude-sonnet-4-5", "anthropic", "claude-sonnet-4-5")
    );
    adapter.updateList(defaults);
}
```

### ConfigTemplateCache Errors

**Scenario: SharedPreferences corrupted**
```java
public static ConfigTemplate loadTemplate(Context ctx) {
    try {
        SharedPreferences prefs = ctx.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE);
        // Read fields...
        return template;
    } catch (Exception e) {
        Logger.logError(TAG, "Failed to load template: " + e);
        clearTemplate(ctx); // Clear corrupted data
        return null;
    }
}
```

**Scenario: Partial data**
```java
public boolean isValid() {
    // Provider, model, apiKey required
    // tgBotToken and tgUserId optional
    return provider != null && !provider.isEmpty()
        && model != null && !model.isEmpty()
        && apiKey != null && !apiKey.isEmpty();
}
```

**Scenario: Version migration**
```java
private static final int CONFIG_VERSION = 1;

public static ConfigTemplate loadTemplate(Context ctx) {
    SharedPreferences prefs = ctx.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE);
    int version = prefs.getInt("version", 0);

    if (version < CONFIG_VERSION) {
        migrateConfig(ctx, version);
    }

    // Load template...
}
```

## UI/UX Considerations

### Model Selector UX

1. **Loading State:** Show spinner while fetching models
2. **Empty State:** Clear message if no models found
3. **Error State:** Friendly error + retry button
4. **Search Feedback:** Real-time filtering as user types
5. **Selection Feedback:** Highlight selected model before closing

### Config Restore UX

1. **Non-intrusive:** Show as optional dialog, not forced
2. **Clear Action:** "Use Previous" vs "Start Fresh" buttons
3. **Skip if Broken:** If template invalid, don't show dialog
4. **Transparent:** Show what config will be restored

### Update Button UX

1. **Visual Feedback:** Disable during check, show loading
2. **Clear Result:** Toast or banner for success/failure
3. **Accessible:** Easy to spot but not intrusive
4. **Consistent:** Same style as other navigation buttons
